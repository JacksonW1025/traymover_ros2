# ROS To Driver 串口通信协议解析

本文档基于底层的 C++ 数组定义（`boost::array<uint8_t, 40>`），详细描述了总长度为 **40 字节** 的定长串口通信协议数据帧结构，以及预定义的静态控制帧。

## 一、 数据帧结构详解

> **注意：** > 1. 以下表格中的字节位置完全对应源定义的 0-39 索引。
> 2. 协议主要采用 **大端模式 (Big-Endian)** 传输 16 位整数（高位在前，低位在后）。

| 字节位 (Byte) | 字段名称 (Field Name)                     | 长度  | 含义说明与具体取值 (Description & Values)                    |
| :------------ | :---------------------------------------- | :---- | :----------------------------------------------------------- |
| **0 - 1**     | `Head` (帧头)                             | 2字节 | 固定为 `0x7F, 0x7F`。                                        |
| **2**         | `Lenth` (数据长度)                        | 1字节 | 对应数据包总长度 `0x28` (十进制 40)。                        |
| **3**         | `Serial ID Cmd` (命令ID)                  | 1字节 | 接收 Send Type 数据 (控制指令类型)。                         |
| **4**         | `Move Cmd` (运动控制指令)                 | 1字节 | `0x00`: Forward (前进)<br>`0x01`: backward (后退)<br>`0x02`: Turn Left (左转)<br>`0x03`: Turn Right (右转)<br>`0x04`: Stop (停止) |
| **5 - 6**     | `Distance` (距离)                         | 2字节 | 字节 5：高 8 位<br>字节 6：低 8 位                           |
| **7**         | `data` (系统模式数据)                     | 1字节 | `0x00`: Auto Charge Mode (自动充电模式)<br>`0x01`: Manual Mode (手动模式)<br>`0x02`: Set Out Charge Mode (退出充电模式) |
| **8**         | `TOF Range Sensor Switch` (TOF测距开关)   | 1字节 | 控制 TOF 传感器的屏蔽与启用：<br>• 0x00: 左 TOF：0 屏蔽，1 启用<br>• 0x02: 中 TOF：0 屏蔽，1 启用<br>• 0x03: 右 TOF：0 屏蔽，1 启用 |
| **9 **        | 空白                                      | 1字节 | 无                                                           |
| **10 - 11**   | `Vel_x` (X轴线速度)                       | 2字节 | 字节 10：高 8 位<br>字节 11：低 8 位                         |
| **12 - 13**   | `Vel_th` (Z轴角速度)                      | 2字节 | 字节 12：高 8 位<br>字节 13：低 8 位                         |
| **14 - 15**   | `Left Motor Scale` (左电机比例)           | 2字节 | 字节 14：高 8 位<br>字节 15：低 8 位                         |
| **16 - 17**   | `Right Motor Scale` (右电机比例)          | 2字节 | 字节 16：高 8 位<br>字节 17：低 8 位                         |
| **18**        | `update motor chip` (更新驱动芯片)        | 1字节 | 用于更新电机驱动芯片。                                       |
| **19 - 20**   | `Motor Filter` (电机滤波)                 | 2字节 | 字节 19：高 8 位<br>字节 20：低 8 位                         |
| **21**        | `alarm led switch` (报警灯开关)           | 1字节 | 控制报警灯状态。                                             |
| **22**        | `Infrared Range Sensor Switch` (红外开关) | 1字节 | 控制红外传感器的屏蔽与启用：<br>• 0x00: 左 Infrared：0 屏蔽，1 启用<br>• 0x01: 中 Infrared：0 屏蔽，1 启用<br>• 0x02: 右 Infrared：0 屏蔽，1 启用 |
| **23**        | `Left Infrared Range Sensor`              | 1字节 | 左侧红外参考值：设置左侧参考值，实际测量值小于该值即触发。   |
| **24**        | `Middle Infrared Range Sensor`            | 1字节 | 中间红外参考值：设置中间参考值，实际测量值小于该值即触发。   |
| **25**        | `Right Infrared Range Sensor`             | 1字节 | 右侧红外参考值：设置右侧参考值，实际测量值小于该值即触发。   |
| **26**        | `Wheel Diameter` (轮子直径)               | 1字节 | 设定轮子的直径。                                             |
| **27 - 28**   | `gear reduction 低位` (减速比低位)        | 2字节 | 电机的编码数低位。                                           |
| **29 - 30**   | `wheel track` (轮距)                      | 2字节 | 车辆的轮距。                                                 |
| **31**        | `tick meter ratio` (里程计比例)           | 1字节 | tick meter 比例参数。                                        |
| **32**        | `蜂鸣器` (Buzzer)                         | 1字节 | `0`: 不响<br>`1`: 响                                         |
| **33**        | `到达点位信号` (Arrival Signal)           | 1字节 | 用于给外部机器信号：<br>`0`: 未到达<br>`1`: 到达<br>到达后给pb15引脚高电平 |
| **34 - 35**   | `gear reduction 高位` (减速比高位)        | 2字节 | 电机的编码数 高位。                                          |
| **36**        | `顶升状态` (Jack Status)                  | 1字节 | `0`: 无操作<br>`1`: 上升<br>`2`: 下降                        |
| **37**        | `check` (校验码)                          | 1字节 | **校验规则**：前面所有数据相加后取反，再加 `0x01`。(即所有前置数据的求和补码) |
| **38 - 39**   | `Tail` (帧尾)                             | 2字节 | 固定为 `0x0D, 0x0A` (即 `\r\n`)。                            |

---

一个正确的前进示例参考，他可以驱动底盘向前运动：7F 7F 28 02 00 00 00 00 00 00 01 90 00 00 00 01 00 01 00 00 00 01 00 14 14 14 AA 10 00 01 C7 0A 00 00 00 00 00 7C 0D 0A
