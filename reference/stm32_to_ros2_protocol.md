# STM32 回传 ROS2 协议表解析汇总

> 根据你提供的图片人工整理而成。  
> 说明：
> 1. 原图中的 `MOTRO` 统一按规范更正为 `MOTOR`；其余英文拼写做了必要规范化，例如 `Lenth -> Length`、`Fileter -> Filter`。  
> 2. 个别单元格边界较模糊时，我按表格布局做了整理；这类地方会在备注中说明。  
> 3. 从图上看，这几类帧都按 **0~39** 共 **40 字节** 排布。  

## 一、通用帧格式

从多张图里能看出的公共部分如下：

| 字节索引 | 字段 | 长度 | 说明 |
|---|---:|---:|---|
| 0~1 | Head | 2字节 | 固定为 `0x7F 0x7F` |
| 2 | Length | 1字节 | 固定为 `0x28` |
| 3 | Serial ID Cmd | 1字节 | 多数表中写为 `Send_Type`；少数异常帧另有定义 |
| 4 | Status | 1字节 | 成功帧通常为 `Send_Success (0x60)`；异常帧为 `Send_Error (0x80)` |
| 37 | check | 1字节 | **前面所有数据相加后取反，再加 `0x01`** |
| 38~39 | Tail | 2字节 | 固定为 `0x0D 0x0A` |

可写成：

```text
check = (~sum(byte[0..36]) + 0x01) & 0xFF
```

### 2) Byte 34 位定义

> 图中第二列用 `0x00 ~ 0x07` 标出了 byte 34 的各 bit 含义，下面按**从上到下**整理。

| bit（按图中 0x00~0x07） | 含义 | 取值说明 |
|---|---|---|
| 0x00 | 左传感器距离 | `0=未触发`，`1=触发` |
| 0x01 | 中传感器距离 | `0=未触发`，`1=触发` |
| 0x02 | 右传感器距离 | `0=未触发`，`1=触发` |
| 0x03 | 左传感器开关 | `0=屏蔽`，`1=启用` |
| 0x04 | 中传感器开关 | `0=屏蔽`，`1=启用` |
| 0x05 | 右传感器开关 | `0=屏蔽`，`1=启用` |
| 0x06 | 电机使能 | `0=接通`，`1=关闭` |
| 0x07 | 防撞条 | `0=未触发`，`1=触发` |

## 七、可直接用于代码实现的简化索引

### 1) 成功帧公共头尾

```text
byte[0]  = 0x7F
byte[1]  = 0x7F
byte[2]  = 0x28
byte[4]  = 0x60   // Send_Success
byte[37] = checksum
byte[38] = 0x0D
byte[39] = 0x0A
```

### 2) 错误帧公共头尾

```text
byte[0]  = 0x7F
byte[1]  = 0x7F
byte[2]  = 0x28
byte[4]  = 0x80   // Send_Error
byte[37] = checksum
byte[38] = 0x0D
byte[39] = 0x0A
```

### 3) 最值得单独处理的字段

- `MSG_ID_MOTOR_DATA = 2`
  - 编码器：`11~14`
  - TOF 左中右：`15~16`、`19~20`、`21~22`
  - 速度：`29~32`
  - 驱动模式：`33`
  - 红外/碰撞综合 bitfield：`34`

- `MSG_ID_MOTOR_STATUS = 3`
  - 温度：`5~6`
  - 电流：`7~8`
  - 速度：`10~13`
  - TOF 使能字：`19`
  - 左右标定量：`20~23`
  - APP 版本：`25~26`
